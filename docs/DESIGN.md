# 今日拍板 · 本地探索 Agent — 设计文档

> 赛题 06「本地探索：周末闲时活动规划」｜接受一句自然语言目标，输出可执行完整方案并自动完成下单/预约。

## 一、产品核心创新：从"搜索推荐"到"帮你把事做完"

| 创新点 | 解决的真实痛点 |
|---|---|
| **① 隐性约束推断** | 用户只说"带老婆孩子玩"，系统主动推断出"孩子5岁→需亲子友好、餐厅要儿童椅""老婆减肥→优先低卡餐厅""别太远→收紧通勤半径"。不是过滤，是替用户想到他没想到的。 |
| **② 接力模式（行业首创）** | 小明把手机递给老婆/朋友的那一刻，界面自动切换"视角"，用对方最关心的维度重新渲染方案并解释"专门为你考虑"，把群体决策摩擦降到一键拍板。 |
| **③ 确认即执行** | 用户点确认后，后台**并行**完成餐厅预约 / 购票 / 蛋糕鲜花下单，并生成可转发行程卡。 |

## 二、Planning 策略

```
自然语言目标
   │
   ▼  intent.py —— 意图解析（规则引擎兜底 + LLM 可选增强）
ParsedIntent { 场景, 成员画像, 人数, 时长, 位置 }
   │
   ▼  planner.infer_constraints —— 隐性约束推断
[Constraint{key,value,source,reason}]  每条都带"为什么"，用于向用户解释
   │
   ▼  planner.build_plans —— 多主题方案生成
   │   家庭场景：亲子优先 / 减脂友好
   │   朋友场景：性价比聚会 / 出片体验
   │   每套方案 = 活动 → 正餐 → 附加活动（含时间排布）
   ▼
Plan[]  （第一套为推荐）
```

**为什么用"规则兜底 + LLM 可选"混合架构**：演示现场零依赖、零延迟、结果可复现；配置 `LLM_API_KEY` 后意图解析自动切换真实大模型增强泛化能力。两全。

**多方案而非单方案**：群体出行偏好不一，给 2 套不同主题让用户/家人对比选择，比强推一套更符合真实决策。

## 三、工具调用链

```
规划阶段                          执行阶段（用户确认后并行）
─────────                        ──────────────────────────
poi.search_activities ┐          restaurant.reserve ──┐ 满位
restaurant.search_restaurants ┘   ticket.buy_tickets ──┤ 售罄  → asyncio.gather 并行
                                  order.place_order  ──┘
                                          │
                                  itinerary.build_itinerary → 可分享行程卡
```

所有工具均为 Mock API（`tools/` + `data/mock_db.py`），模拟真实接口：含网络延迟、库存/满位等动态状态，使异常处理可被真实触发。

| 接口 | 输入 | 输出 |
|---|---|---|
| `POST /api/plan` | 自然语言目标 | 意图解析 + 约束推断 + 多方案 + session_id |
| `POST /api/relay` | 方案 + 受众(spouse/friends) | 接力视角卡（重渲染的关注点 + 一键反馈） |
| `POST /api/execute` | session_id + 方案 + 额外动作 | 并行执行结果 + 行程卡 |

## 四、异常处理机制

| 异常 | 触发条件（Mock） | 处理策略 |
|---|---|---|
| **餐厅满位** | 排队 > 35 分钟 | 自动在同约束下搜索备选餐厅改订，标记 `fallback` 并说明原因；无备选则 `failed` 提示手动改约 |
| **门票售罄** | 库存 = 0（如蹦床馆） | 标记 `failed`，提示现场购票或更换活动；**不阻塞**其它动作 |
| **单动作失败** | 任一工具异常 | `asyncio.gather` 并行执行互不影响，失败项独立标记，整体仍产出行程卡 |
| **LLM 失败/未配置** | 无 key 或调用异常 | 静默回退规则引擎，功能不降级 |
| **会话过期** | execute 时 session 丢失 | 用方案反推最小意图兜底，仍可执行 |

降级全程对用户透明：界面用 ✅成功 / 🔄已改订 / ⚠️需确认 三态清晰呈现。

## 五、技术栈与可扩展性

- **后端** Python + FastAPI，分层清晰：`models`(契约) / `data`(mock库) / `tools`(API层) / `agent`(理解·规划·接力·执行) / `main`(路由)
- **前端** 原生 HTML/CSS/JS 单页，美团风格移动端 UI，零构建依赖
- **可扩展** Mock API 与真实美团 POI/交易接口同构，替换 `tools/` 实现即可对接生产；约束推断规则可持续沉淀为知识库。
